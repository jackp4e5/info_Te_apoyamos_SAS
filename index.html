Análisis de Proyecto web - Firma de Abogados "Te Apoyamos SAS"

La firma de abogados “Te Apoyamos SAS” busca ampliar su cobertura de servicios mediante el uso de una herramienta web que le permita posicionarse en el mercado global y brindar un espacio digital para sus clientes y profesionales del gremio. Actualmente, la organización carece de un sistema centralizado en línea que dé a conocer información relevante, permita la gestión de profesionales y facilite el seguimiento de los procesos legales que adelanta.

La ausencia de esta plataforma genera limitaciones en visibilidad, gestión documental y comunicación con potenciales clientes y profesionales interesados en vincularse. De esta manera, la solución web debe responder a tres necesidades fundamentales: información institucional, gestión de talento humano y gestión de procesos legales.

Requerimientos

Requerimientos funcionales

  • Página de inicio con información de la firma, trayectoria, reconocimientos y alianzas
  • Sección de servicios ofertados con descripción detallada.
  • Espacio para noticias de interés legal.
  • Perfiles de los profesionales vinculados a la firma.
  • Módulo de registro de hojas de vida para aspirantes.
  • Validación de aceptación o rechazo de las hojas de vida registradas.
  • Módulo de gestión de procesos legales: registro de datos del cliente, tipo de proceso y avances.
  • Sección de contacto con formulario y ubicación de oficinas en el territorio nacional.

Requerimientos no funcionales

  • Seguridad en el manejo de la información (certificados SSL, autenticación de usuarios).
  • Respaldo y recuperación de datos.
  • Diseño responsive para garantizar accesibilidad desde cualquier dispositivo.
  • Facilidad de uso con una interfaz clara e intuitiva.
  • Escalabilidad para permitir futuras ampliaciones del sistema.

Actores involucrados y Información a registrar

    Actores involucrados

  • Cliente general: persona interesada en conocer la firma, consultar servicios o establecer contacto.
  • Profesional aspirante: abogado que registra su hoja de vida para aplicar a la firma.
  • Profesional adscrito: abogado que pertenece a la firma y gestiona procesos legales.
  • Administrador del sistema: encargado de validar hojas de vida, aceptar/rechazar aspirantes y supervisar la gestión de procesos y datos.
  • Visitante general: usuario que accede al sitio únicamente para leer información institucional y noticias.

    Información a registrar

  • Información institucional: trayectoria, reconocimientos, alianzas, servicios y noticias de interés.
  • Información de profesionales aspirantes: nombre, identificación, formación académica, experiencia laboral, datos de contacto, hoja de vida adjunta.
  • Información de profesionales adscritos: perfil general, experiencia, área de especialidad, ubicación dentro del territorio nacional.
  • Información de clientes y procesos: datos personales del cliente, tipo de proceso legal, descripción del caso, estado y avances.
  • Información de contacto: consultas enviadas por los clientes (nombre, correo, mensaje).

Frameworks y herramientas para diseño web usable y responsive

    Frameworks

  • Bootstrap

    Es uno de los frameworks más usados para diseño responsive. Ofrece un sistema de rejillas (grid system) que facilita adaptar el contenido a pantallas de distintos tamaños, además de componentes ya diseñados (botones, menús, formularios).

  • Tailwind CSS

    Es un framework de CSS moderno que permite personalizar estilos de forma rápida y flexible. Favorece la creación de interfaces limpias y responsivas mediante clases utilitarias, evitando escribir mucho CSS desde cero

  • Foundation

    Menos conocido que Bootstrap, pero muy potente. Ofrece componentes accesibles y adaptativos, además de buenas prácticas en usabilidad y accesibilidad web.

  • Materialize

    Basado en las guías de diseño de Google Material Design. Permite crear páginas modernas y responsivas con componentes visuales consistentes.

  • React (librería) y Vue.js / Angular (frameworks)

    permiten crear aplicaciones web más usables, porque hacen que la interacción con el usuario sea más fluida (sin recargar la página completa, manejo de componentes, estados, etc.).

  • Para el diseño responsive, normalmente se combinan con frameworks de CSS como Bootstrap o Tailwind. Ejemplo: React + Tailwind o Vue + BootstrapVue.

    Bootstrap, Tailwind, Foundation, Materialize, estan enfocados en responsive design y usabilidad visual.

    React, Vue.js, Angular estan enfocados en usabilidad funcional e interacción dinámica. Por lo tanto, estos no son específicamente frameworks de diseño responsive, contribuyen a la usabilidad y al desarrollo de interfaces modernas.
